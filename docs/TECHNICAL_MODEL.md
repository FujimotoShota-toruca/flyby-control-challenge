# Flyby Control Challenge 数値・物理モデル解説

## 1. 本書の目的

本書は、`Flyby Control Challenge` で使用している物理計算、誤差分布、安全判定、確率表示、点数計算を、実装と対応する数式として整理した技術資料である。

本アプリは、はやぶさ2・トリフネフライバイを題材にした**非公式の教育用簡略モデル**である。実軌道、実際の航法解、実機の噴射性能、JAXA/ISASの運用判断を再現するものではない。

本書は、文書作成時点の `src/lib/flybyModel.ts`、`src/lib/scoring.ts`、`src/lib/missionPresets.ts` を基準にしている。`docs/SPEC.md` に残る初期案・旧スコア案より、現在の実装を優先して記述する。

モデルの目的は、次の関係を体験できるようにすることである。

1. 小さな速度変更 `ΔV` でも、時間が経つと通過位置が大きく変わる。
2. 早い時期の修正は大きく効き、遅い時期の修正は細かな仕上げに向く。
3. 大きな `ΔV` は通過位置を大きく動かす一方、噴射のずれも大きくする。
4. 予測中心だけでなく、予想範囲全体を危険領域から外す必要がある。
5. 観測距離、安全性、`ΔV` 使用量の間にはトレードオフがある。

## 2. モデル全体の位置付け

### 2.1 物理的な考え方を使っている部分

- `ΔV` の積み重ねをベクトルの大きさで評価する。
- `ΔV` からBプレーン上の通過位置変化を、線形感度行列で近似する。
- 実行誤差の共分散を、感度行列によってBプレーン共分散へ写像する。
- 共分散行列の固有値・固有方向から誤差楕円を作る。
- 予測分布を二次元正規分布として扱う。
- 予測楕円外縁と危険円の幾何学的な重なりで安全を判定する。

### 2.2 教育用・ゲーム用に設定している部分

- 感度行列の具体的な数値。
- 光学観測後の中心位置補正量。
- 航法誤差共分散と、観測後に残る第1回噴射誤差の割合。
- 噴射誤差率と、大きな噴射に対する二次ペナルティ。
- 危険領域、観測コース、各スコアの境界と配点。
- 危険時に合計点を最大59点へ制限する規則。

これらは、実ミッションの確定値ではなく、スマートフォンで操作できる教材として、2回の修正の役割と安全・観測のトレードオフが見えやすくなるよう調整した値である。

### 2.3 「根拠」の種類

本書では、数値や式の根拠を次の3種類に分ける。

| 種類 | 例 | 本アプリでの扱い |
|---|---|---|
| 物理・数学の一般式 | `Δr ≈ Δv t`、`P_B = H P_ΔV H^T`、共分散の固有値分解 | 考え方と計算形式の根拠として使用 |
| 公開情報を参考にした代表値 | トリフネの大きさ、約1 km級フライバイ、約5 km/s級の相対速度 | 距離感・教材説明の参考として使用 |
| 教材・ゲーム用の調整値 | 感度行列、誤差率、危険半径、3〜5 km満点帯、配点 | 学習目標と操作性を満たすよう設定 |

したがって、数式の形が物理・数学に基づいていても、代入している全パラメータが実ミッション由来とは限らない。

## 3. 座標・記号・単位

本アプリは、フライバイ時の通過位置を二次元Bプレーン上で表す。

```txt
B = [B_y, B_z]^T  [km]
ΔV = [ΔV_y, ΔV_z]^T  [m/s]
```

| 記号 | 意味 |
|---|---|
| `B_y`, `B_z` | Bプレーン上の通過位置 |
| `ΔV_y`, `ΔV_z` | 操作する速度変更量 |
| `H_i` | 第 `i` 回修正の感度行列 |
| `P` | 二次元誤差共分散行列 |
| `d_CA` | 最終予測中心の中心距離 |
| `r_danger` | 危険領域半径 |
| `m_3σ` | 3σ予想楕円による安全余裕 |

Bプレーン原点はトリフネ中心とする。アプリ内の `d_CA` は、最終予測中心のBプレーン原点からの距離である。

```math
d_{CA} = \sqrt{B_y^2 + B_z^2}
```

これは、重力偏向等を解いた厳密な最近接距離ではなく、Bプレーン上の中心距離を最近接距離として扱う簡略化である。

## 4. ΔVによる通過位置の変更

### 4.1 基本的な物理背景

短い噴射で速度が `Δv` だけ変化し、その後 `t` 秒間飛行すると、最も単純な一次近似では位置差は次のように増える。

```math
\Delta r \approx \Delta v \, t
```

`Δv` を `m/s`、結果を `km` で表す場合は、次の換算になる。

```math
\Delta r_{km} \approx \frac{t_{sec}}{1000}\Delta v_{m/s}
```

例えば、`T-24 h = 86,400 s` で `0.1 m/s` の速度差を与えると、単純な直線近似では約 `8.64 km` の位置差になる。

この「早く修正すると、小さなΔVでも大きく効く」という関係を、本アプリでは感度行列で表現している。

### 4.2 感度行列

各修正によるBプレーン移動量を、次の線形モデルで計算する。

```math
\Delta B_i = H_i \Delta V_i
```

標準シナリオの感度行列は次のとおりである。

```math
H_1 =
\begin{bmatrix}
86.4 & 12.0 \\
-7.0 & 78.0
\end{bmatrix}
\quad [km/(m/s)]
```

```math
H_2 =
\begin{bmatrix}
28.0 & 6.0 \\
-3.5 & 25.0
\end{bmatrix}
\quad [km/(m/s)]
```

対角成分は、各方向の `ΔV` が主に同じ方向のBプレーン座標を動かす効果を表す。非対角成分は、軌道幾何によって一方の方向の噴射がもう一方の座標にも影響する様子を簡略表現する。

`H_1` は `H_2` より大きく、第1回修正の方が通過位置を大きく動かせる。これらは実軌道から導出した状態遷移行列ではなく、`Δr ≈ Δv t` を基礎イメージとした教育用感度行列である。

### 4.3 2回の修正と観測更新

初期中心は次の値である。

```math
B_0 =
\begin{bmatrix}
8 \\
-12
\end{bmatrix}
\quad [km]
```

第1回修正後の中心：

```math
B_1 = B_0 + H_1 \Delta V_1
```

追加の光学観測後の中心：

```math
B_{nav} = B_1 + L_1 z_{obs}
```

ここで、`P_1 = L_1 L_1^T` を満たす `L_1` は第1回修正後共分散のコレスキー因子、`z_obs` は標準二次元正規分布から抽選した値である。抽選結果は3σ範囲内に制限する。

最終中心：

```math
B_f = B_{nav} + H_2 \Delta V_2
```

追加観測によって、第1回の広い予測分布の中から、今回のチャレンジで判明した中心位置が明らかになる。第1回ΔVを実行するたびに新しい点を抽選するが、第2回運用中は固定し、操作中に目標が動かないようにする。実際の画像航法や軌道決定計算を行っているわけではない。

## 5. ΔVバジェット

各回の `ΔV` 使用量は、二次元ベクトルのノルムで計算する。

```math
\lVert \Delta V_i \rVert =
\sqrt{\Delta V_{y,i}^2 + \Delta V_{z,i}^2}
```

総使用量と残量：

```math
\Delta V_{used} =
\lVert \Delta V_1 \rVert + \lVert \Delta V_2 \rVert
```

```math
\Delta V_{remaining} =
\max(0,\ 0.30 - \Delta V_{used})
```

第1回・第2回を合わせた総バジェットは `0.30 m/s` である。入力がバジェットを超える場合、希望した `ΔV` ベクトルの方向を保ったまま、残量に収まる大きさへ縮小する。

## 6. ΔV実行誤差モデル

### 6.1 誤差の考え方

噴射指令どおりに完全な `ΔV` を出せるとは限らない。本アプリでは、噴射方向に沿った誤差と、その直交方向の誤差を分ける。

```math
m = \lVert \Delta V \rVert
```

大きな噴射に対する追加ペナルティ：

```math
q = k_{large} m^2
```

噴射方向に沿った1σ誤差：

```math
\sigma_{\parallel}
= \sigma_{floor} + k_{prop}m + q
```

直交方向の1σ誤差：

```math
\sigma_{\perp}
= 0.7\sigma_{floor} + k_{cross}m + 0.45q
```

標準パラメータ：

| パラメータ | 値 |
|---|---:|
| 固定誤差下限 `σ_floor` | `0.002 m/s` |
| 第1回比例誤差率 | `0.16` |
| 第2回比例誤差率 | `0.10` |
| 直交方向比例誤差率 | `0.08` |
| 大噴射ペナルティ係数 `k_large` | `0.35` |

第2回修正は第1回修正より比例誤差率を小さくし、追加観測後に行う精密修正として表現する。

二次項 `m^2` は、非常に大きな指令を一度に行うと誤差が急増するゲーム性を作るための教育用項である。実機の誤差モデルから導出したものではない。

### 6.2 ΔV誤差共分散

`ΔV` 方向の角度を次のように置く。

```math
\theta = \operatorname{atan2}(\Delta V_z,\Delta V_y)
```

回転行列：

```math
R(\theta) =
\begin{bmatrix}
\cos\theta & -\sin\theta \\
\sin\theta & \cos\theta
\end{bmatrix}
```

`ΔV` 座標系の誤差共分散をBプレーン座標軸へ回転する。

```math
P_{\Delta V}
= R(\theta)
\begin{bmatrix}
\sigma_{\parallel}^2 & 0 \\
0 & \sigma_{\perp}^2
\end{bmatrix}
R(\theta)^T
```

このため、`ΔV` の大きさだけでなく方向によっても、最終的な誤差楕円の向きが変化する。

## 7. Bプレーン共分散の伝播

線形変換 `ΔB = H ΔV` に対し、共分散は次式で写像する。

```math
P_{B,exec,i} = H_i P_{\Delta V,i} H_i^T
```

### 7.1 第1回修正後

```math
P_1 = P_{nav,1} + P_{B,exec,1}
```

第1回修正前の基礎航法共分散：

```math
P_{nav,1} =
\begin{bmatrix}
4.7151358142 & 1.5640074047 \\
1.5640074047 & 2.0904197413
\end{bmatrix}
\quad [km^2]
```

これは3σで、長い方向 `7.0 km`、短い方向 `3.5 km`、角度約 `25°` の楕円になるよう設定している。

### 7.2 追加観測後・第2回修正前

```math
P_{pre2} = P_{nav,2} + \rho P_{B,exec,1}
```

```math
\rho = 0.03
```

追加観測後の基礎航法共分散：

```math
P_{nav,2} =
\begin{bmatrix}
0.4007727405 & -0.1199870205 \\
-0.1199870205 & 0.1147828151
\end{bmatrix}
\quad [km^2]
```

これは3σで、長い方向 `2.0 km`、短い方向 `0.8 km`、角度約 `-20°` の楕円になるよう設定している。

基礎楕円だけを比べると、長い方向は `7.0 / 2.0 = 3.5` 倍、短い方向は `3.5 / 0.8 = 4.375` 倍細くなる。3σ面積は約 `76.97 km²` から約 `5.03 km²` となり、追加観測後は初期の約 `6.5%`、すなわち約 `93.5%` 縮小する設定である。

この大幅な縮小は、「遠方では不確かだが、追加の光学観測で通過位置を絞り込める」という運用上の傾向を強調する教育用設定である。公開資料から、トリフネフライバイにおける観測前後のBプレーン3σ長半径・短半径がこの比率になると確認した値ではない。実際の縮小率は、観測時刻、画像上の検出精度、観測回数、幾何条件、軌道決定手法などに依存する。

`ρ = 0.03` は、第1回噴射によって増えた**共分散の3%**を追加観測後にも残す設定である。標準偏差の3%を意味するものではない。

### 7.3 最終結果

```math
P_f =
P_{nav,2}
+ \rho P_{B,exec,1}
+ P_{B,exec,2}
```

第2回 `ΔV` を大きくすると `P_{B,exec,2}` が増えるため、最終中心を動かせる一方で最終予想楕円も広がる。

## 8. 共分散から3σ楕円への変換

二次元共分散行列を次のように置く。

```math
P =
\begin{bmatrix}
a & b \\
b & d
\end{bmatrix}
```

固有値は次式で求める。

```math
\lambda_{major,minor}
= \frac{a+d}{2}
\pm
\sqrt{
\left(\frac{a-d}{2}\right)^2+b^2
}
```

1σの長半径・短半径：

```math
\sigma_{major} = \sqrt{\lambda_{major}}
```

```math
\sigma_{minor} = \sqrt{\lambda_{minor}}
```

楕円の向き：

```math
\phi =
\frac{1}{2}\operatorname{atan2}(2b,\ a-d)
```

画面に表示する3σ楕円の半径：

```math
r_{major,3\sigma} = 3\sigma_{major}
```

```math
r_{minor,3\sigma} = 3\sigma_{minor}
```

3σ楕円面積：

```math
A_{3\sigma}
= \pi (3\sigma_{major})(3\sigma_{minor})
= 9\pi\sigma_{major}\sigma_{minor}
```

本アプリでは、描画、安全判定、面積表示をすべて3σへ統一している。二次元正規分布において「3σ楕円内の確率」は、一次元の3σと完全に同じ割合ではないが、団員向けには「ほとんどの通り道が入る予想範囲」と説明する。

マップ上では、中心ほど濃く、3σ外縁へ向かって透明になる楕円グラデーションとして表示する。これは分布に中心と外側の違いがあることを直感的に伝えるための表現である。安全判定に使う3σ外縁は、濃淡を邪魔しない細い破線で重ねて表示する。安全判定と面積計算には、この3σ外縁を使用する。

## 9. 安全判定

### 9.1 中心点による距離

最終予測中心の距離：

```math
d_{CA} = \sqrt{B_{y,f}^2 + B_{z,f}^2}
```

中心点だけによる単純な安全余裕：

```math
m_{center} = d_{CA} - r_{danger}
```

ただし、実際の安全スコアは中心点だけでなく、最終分布160標本の距離ポテンシャル平均を使用する。3σ楕円は安全条件と59点上限の判定に使用する。

### 9.2 3σ楕円による安全余裕

危険領域半径は次の値である。

```math
r_{danger} = 0.75\ km
```

3σ安全余裕は、回転した3σ楕円外縁とトリフネ中心の最短距離から危険領域半径を引いて求める。

```math
m_{3\sigma}
= \min_{p \in E_{3\sigma}}\lVert p\rVert
- r_{danger}
```

判定：

```txt
m_3σ >= 0  : 3σ楕円全体が危険領域外
m_3σ < 0   : 3σ楕円が危険領域と重なる
```

実装では、楕円外縁を4,096点に分割して最短距離を数値的に求める。楕円がトリフネ中心を内包する場合は、負の安全余裕として `-0.75 km` を返す。この場合、負値の大きさは厳密な食い込み量ではなく、「安全条件未達」を明確に示すための値である。

### 9.3 表面からの距離目安

トリフネの最大半径目安を `0.42 km` として、表面からの距離目安を表示する。

```math
d_{surface} = d_{CA} - 0.42
```

形状モデルを使用していないため、これは方向によらない概算値である。

## 10. 確率表示

最終中心 `B_f` と最終共分散 `P_f` から、二次元正規分布に従う点を160個生成する。

```math
x_j = B_f + L z_j
```

ここで、`L` は `P_f = LL^T` を満たすコレスキー因子、`z_j` は標準正規分布に対応する二次元点である。

第1回ΔV実行時に乱数シードを作り、そのシードから決定的なBox-Muller変換で160点を生成する。同じチャレンジ中は同じ操作に対して同じ確率を表示するが、最初から再挑戦した場合や第1回ΔVを再実行した場合は、追加観測結果が変化する。

各確率は、条件を満たした点数の割合として計算する。

```math
\hat{p}(A) =
\frac{\#\{x_j \in A\}}{160}
```

| 表示・内部値 | 距離条件 |
|---|---|
| 危険領域へ入る確率 | `d < 0.75 km` |
| おすすめ領域へ入る確率 | `1 <= d <= 10 km` |
| 近くで見られる確率 | `1 <= d < 3 km` |
| 安全に見られる確率 | `3 <= d <= 10 km` |

160点による標本比率なので、確率表示の最小刻みは `1 / 160 = 0.625%` である。これは厳密な衝突確率計算ではなく、分布と距離帯の関係を理解するための補助表示である。

## 11. 距離帯と教材上の意味

| 半径 | 教材上の意味 |
|---|---|
| `0.225 km` | トリフネ等価半径の参考値 |
| `0.42 km` | 表面距離計算に使う最大半径目安 |
| `0.75 km` 未満 | 安全バッファ込み危険領域 |
| `1〜3 km` | 近くで見られる領域 |
| `3〜10 km` | 安全に見られる領域 |
| `3〜5 km` | 安全性を重視した観測距離の例 |
| `30 km` | Bプレーン全体表示の距離目安 |

ゲーム内で3〜5 kmを満点観測コースにしたのは、予想楕円全体の安全を確保しながら、十分近い観測コースを作る課題にするためである。

実ミッションで検討される中心距離約1 kmのフライバイは、本ゲームの満点帯よりかなり近い。ゲームを通じて3〜5 kmへ安全に入れる難しさを体験した後に、実ミッションが高精度な航法と慎重な運用によって、さらに攻めた距離を目指していることを説明できる設計としている。

## 12. 点数計算

### 12.1 合計点

点数は100点満点である。

```math
S_{raw}
= S_{science}
+ S_{safety}
```

| 項目 | 最大点 |
|---|---:|
| 科学的意義得点 | 50 |
| 安全得点 | 50 |

第1回修正で近づけた量、第2回修正による改善量、残った `ΔV` 量は得点へ加えない。同じ最終分布へ到達した作戦は、操作手順が異なっても同じ得点になる。

### 12.2 距離ポテンシャル

各予測点の中心距離を `r` とする。安全性と科学的意義が反対方向へ動くよう、それぞれ独立したポテンシャルを定義する。

科学的意義ポテンシャルは、実ミッションで検討される中心距離約1 kmを頂点とするガウス形とする。

ただし、中心距離がトリフネ本体の最大半径 `0.42 km` 未満となる標本は衝突コースとみなし、観測データをダウンリンクできないため科学的意義ポテンシャルを `0` とする。

```math
U_{science}(r)
=
\exp
\left(
-\frac{(r-1)^2}{2(2.5)^2}
\right)
```

安全ポテンシャルでは、まず距離を危険半径0.75 kmから安全得点が最大になる3 kmの間へ正規化する。

```math
x(r) =
\operatorname{clamp}
\left(
\frac{r-0.75}{3.0-0.75},0,1
\right)
```

次に、両端で滑らかになるsmoothstep関数を使用する。

```math
U_{safety}(r)
= x(r)^2(3-2x(r))
```

安全ポテンシャルは約1.875 kmで変曲し、0.75 km付近では低く、3 km以上で最大となる。一方、科学的意義ポテンシャルは1 km付近で最大となる。このため、同じ距離で両方を同時に最大化することはできず、どこまで近づくかが作戦上の選択になる。

### 12.3 分布全体の科学的意義得点・安全得点

最終分布の160標本を `x_j`、各中心距離を `r_j = ||x_j||` とする。

```math
S_{science}
= 50 \frac{1}{160}
\sum_{j=1}^{160}
U_{science}(r_j)
```

```math
S_{safety}
= 50 \frac{1}{160}
\sum_{j=1}^{160}
U_{safety}(r_j)
```

中心点だけでなく、楕円の大きさ・向きと、危険側・遠方側へ広がった部分も点数へ反映される。

科学的意義と安全性は反対方向へ動くため、100点を取れる最終分布は存在しない。予想分布が一点へ集まる理想状態で距離だけを最適化しても、最大は中心距離約2.78 kmにおける約87.47点である。実際の分布には広がりがあるため、到達可能な得点はこれ以下になる。100点満点という尺度は、唯一の正解を示すものではなく、教材で設定した評価条件による比較の目安である。

### 12.4 第2回修正による変化の参考表示

第2回修正前と最終結果について、科学的意義得点、安全得点、予想楕円面積を同じ方法で計算し、差分を結果画面へ表示する。これらの差分はミッションスコアへ加点しない。

### 12.5 危険時の合計点上限

最終3σ楕円が危険領域と重なる場合、合計点を最大59点へ制限する。

```math
S_{total} =
\begin{cases}
\min(S_{raw},59) & m_{3\sigma} < 0 \\
S_{raw} & m_{3\sigma} \ge 0
\end{cases}
```

これは物理法則ではなく、「観測コースがよくても、最終安全を満たしていない作戦は合格点にしない」という教材上のルールである。

## 13. 計算例

次の段階的な修正と、あるチャレンジで抽選された追加観測結果を例とする。追加観測後の中心は毎回変わるため、以下は一例である。

```txt
第1回 ΔV = [-0.10, +0.10] m/s
第2回 ΔV = [-0.04, -0.06] m/s
```

第1回によるBプレーン移動量：

```math
H_1\Delta V_1
=
\begin{bmatrix}
86.4 & 12 \\
-7 & 78
\end{bmatrix}
\begin{bmatrix}
-0.10 \\
0.10
\end{bmatrix}
=
\begin{bmatrix}
-7.44 \\
8.50
\end{bmatrix}
km
```

第1回修正後：

```math
B_1 =
\begin{bmatrix}
8 \\
-12
\end{bmatrix}
+
\begin{bmatrix}
-7.44 \\
8.50
\end{bmatrix}
=
\begin{bmatrix}
0.56 \\
-3.50
\end{bmatrix}
km
```

追加観測後：

```math
B_{nav} =
\begin{bmatrix}
-0.84 \\
-1.40
\end{bmatrix}
km
```

第2回によるBプレーン移動量：

```math
H_2\Delta V_2
=
\begin{bmatrix}
-1.48 \\
-1.36
\end{bmatrix}
km
```

最終中心：

```math
B_f =
\begin{bmatrix}
-2.32 \\
-2.76
\end{bmatrix}
km
```

最終中心距離：

```math
d_{CA}
= \sqrt{(-2.32)^2 + (-2.76)^2}
\approx 3.61\ km
```

中心点は近距離の観察に向いた場所へ入っている。ただし、実際の科学的意義得点と安全得点は中心距離だけでは決まらず、最終分布160標本へ距離ポテンシャルを適用した平均値で決まる。さらに、3σ楕円が危険領域へ重なる場合は59点上限が適用される。

## 15. 現在モデル化していない要素

実際のフライバイ運用では、少なくとも次の要素が関係するが、本アプリでは計算していない。

- 太陽・惑星・小惑星の重力を含む軌道積分。
- 双曲線軌道の重力偏向。
- 実際のBプレーン座標系の構築。
- 六次元状態ベクトルと状態遷移行列。
- 観測データからの軌道決定・カルマンフィルタ。
- 小惑星形状と方向に応じた表面距離。
- 姿勢制御、カメラ視野、露光、画像ぶれ。

簡易フライバイ再生の撮像列では、姿勢固定でフライバイするものとして、最接近前のカメラ画像のみトリフネが写る扱いにする。最接近後は、距離が近くてもカメラの向きが追尾しないため視野外と表示する。これは厳密な姿勢運動や視野角計算ではなく、「近ければ常に写るわけではない」という補足を入れるための表示モデルである。
- 通信、電力、熱、運用シーケンス上の制約。
- 実機スラスタの誤差特性。
- 正式な衝突確率・リスク評価。

そのため、本モデルから実ミッションの安全性や必要 `ΔV` を推定してはならない。

## 16. パラメータ調整時の注意

パラメータを変更する場合は、次の関係をまとめて確認する必要がある。

1. `H_1`, `H_2` を大きくすると、小さな `ΔV` で中心を大きく動かせる。
2. 感度行列を大きくすると、同じ実行誤差でもBプレーン楕円が広がる。
3. 噴射誤差率を大きくすると、大きな `ΔV` の不利が強くなる。
4. 基礎航法共分散を大きくすると、`ΔV = 0` でも楕円が広くなる。
5. `ρ` を大きくすると、第1回噴射誤差が第2回運用後にも強く残る。
6. 危険半径を大きくすると、安全条件が厳しくなる。
7. 科学的意義ポテンシャルの幅や安全ポテンシャルの変曲点を変更すると、トレードオフと高得点になる距離帯が変わる。
8. 安全判定や配点を変えた場合、危険時59点上限の意味も再確認する。

変更後は、少なくとも次を確認する。

```powershell
npm.cmd test
npm.cmd run build
```

また、次の代表作戦を実際に操作し、結果を比較する。

- 第1回だけで近づく作戦。
- 第1回で中心近くへ寄せ、第2回で安全へ外す作戦。
- 2回に分けて3〜5 kmへ近づく作戦。
- 観測コースには入るが、3σ楕円が危険領域へ重なる作戦。
- 安全だが遠すぎる作戦。

## 17. まとめ

本アプリの中心モデルは、次の一連の計算で構成される。

```txt
ΔV入力
  ↓
感度行列によるBプレーン中心移動
  ↓
ΔV量・方向に応じた実行誤差
  ↓
共分散伝播と3σ予想楕円
  ↓
中心距離・楕円安全余裕・確率表示
  ↓
最終分布に対する科学的意義・安全ポテンシャルの期待値による採点
```

物理的な考え方を保ちながら、厳密な軌道設計ではなく、団員が「小さなΔVの効果」「不確かさ」「観測と安全のトレードオフ」を操作から理解するためのモデルとして設計している。
