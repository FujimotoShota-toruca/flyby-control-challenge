# MENTOR_CHEATSHEET.md

# Flyby Control Challenge メンター向けチートシート

## 1. 今日のゴール

団員に覚えてほしい中心語は2つです。

```txt
ΔV：探査機の速度を少し変えること。
Bプレーン：小惑星の近くをどのあたりで通るかを見る地図。
```

体験として持ち帰ってほしいことは次の通りです。

```txt
・小さなΔVでも、時間が経つと通過位置が変わる。
・近いほど観測しやすい。
・近づきすぎると危険。
・よい作戦は、観測と安全のバランスで考える。
```

## 2. 基本の声かけ

答えを教えるより、問いを投げてください。

```txt
・ΔVを変えると、Bプレーン上の点はどちらに動いた？
・最近接距離が小さくなると、観測スコアはどう変わった？
・安全スコアはどう変わった？
・この作戦は、観測重視？安全重視？
・次に1つだけ変えるなら、どのスライダーを動かす？
・なぜその作戦にした？
```

## 3. 作戦タイプ別の声かけ

### バランス型

観測と安全のバランスがよい作戦です。

声かけ：

```txt
なぜこの距離にしたか説明してみよう。
観測スコアと安全スコアの両方を見てみよう。
```

### 観測重視型

近くを通って、よく観測しようとする作戦です。

声かけ：

```txt
観測スコアは高いですね。安全余裕は十分あるか確認してみよう。
少し外側を通ると、安全スコアはどう変わるかな？
```

### 安全重視型

安全余裕を大きく取った作戦です。

声かけ：

```txt
安全な作戦ですね。もう少し近づけると観測スコアはどうなるかな？
どのΔVを変えると、少し近づけられそう？
```

### 攻めすぎ型

かなり近い通過位置です。観測には有利ですが、安全余裕が小さいです。

声かけ：

```txt
観測にはよさそうですが、安全余裕も見てみよう。
本物の探査なら、少し余裕を残す作戦も考えます。
```

### 遠すぎ型

安全ですが、トリフネから遠く、観測しにくい作戦です。

声かけ：

```txt
安全ですが、少し遠いですね。
どのスライダーを動かすと、もう少し近づけるかな？
```

### 危険領域

安全条件未達です。破壊的な表現は避けてください。

声かけ：

```txt
この作戦は危険領域に入っています。
安全に観測するため、もう少し外側を通るように調整してみよう。
```

## 4. 実ミッションとの差分

このアプリは非公式の教育用簡略モデルです。実際のはやぶさ2運用を再現するものではありません。

実際の探査では、次のような要素も重要です。

```txt
・実軌道計算
・姿勢制御
・カメラの向き
・画像を使った位置推定
・観測機器の制約
・通信や太陽方向
・運用上の安全判断
```

団員向けには、次の程度の説明で十分です。

```txt
今日のゲームでは、ΔVとBプレーンに集中しました。
本物の探査では、それに加えて、探査機の向き、カメラで見つけること、通信や電力の向きなども考えます。
```

避けたい言い方：

```txt

・これは実際のはやぶさ2の軌道を再現しています。
・一番高い点を取れば正解です。
・近ければ近いほどよいです。
・危険領域に入ると爆発します。
・Bプレーンはただのゲーム画面です。
```

言い換え：

```txt
・これは非公式の教育用シミュレーションです。
・実際の探査で大事な考え方の一部を取り出しています。
・よい作戦は、観測と安全のバランスで考えます。
・Bプレーンは、近くをどこで通るかを見る地図です。
```

## 6. トラブル時の運用

```txt
端末で開けない：
近くの人と2〜3人で1台にする。

全体的に通信が悪い：
講師PC投影で代表操作に切り替える。

操作に迷っている：
まずΔV_yだけ動かして、点がどう動くか見てもらう。

点数だけ見ている：
観測スコアと安全スコアの両方を見るよう促す。

危険領域に入れて遊んでいる：
「では安全に観測する作戦に戻してみよう」と促す。
```

## 7. 進行メモ

推奨構成：

```txt
導入：はやぶさ2・トリフネ・フライバイ・ΔV・Bプレーン
Round 1：自由操作
小さな振り返り
Round 2：条件つきチャレンジ
任意共有
実際はもっと難しい補足
まとめ
```

Round 2のチャレンジ例：

```txt
・安全余裕を2 km以上残す。
・最近接距離を5 km前後にする。
・ΔV合計を小さくする。
・観測重視型と安全重視型を作り比べる。
・一番高い点ではなく、説明できる作戦を作る。
```

## 8. 最後のまとめ例

```txt
今日のゲームでは、ΔVでBプレーン上の通過点を動かして、どこを通るかを考えました。
近いほど観測しやすくなりますが、近づきすぎると危険です。
本物の探査では、さらにカメラの向き、探査機の姿勢、画像を使った位置推定、通信や電力の方向も考えます。
今日やったのは、実際の探査で大事な考え方の一部です。
```
