# DECISIONS.md

# Flyby Control Challenge 設計判断記録

本書は、宇宙少年団の例会「はやぶさ2『トリフネフライバイ』について学ぼう」で使用する非公式教育用Webアプリ `Flyby Control Challenge` の設計判断をまとめたものです。

## FBC-DEC-001：目的

本アプリは、教材を主軸としたシミュレータゲームとする。目的は、団員が `ΔV` と `Bプレーン` を使ってフライバイ通過位置を調整し、観測と安全のトレードオフを体験することである。

実ミッションの厳密再現、SPICE連携、実軌道計算、本格3D宇宙シミュレータ化は目的としない。

## FBC-DEC-002：最低完成条件

v0.1では、`ΔV_y` と `ΔV_z` の操作によってBプレーン上の予測通過点が動き、最近接距離、安全余裕、スコア、作戦タイプ、結果コメントが表示されることを最低完成条件とする。

修正タイミングは `T-12 h` 固定とし、計算は次の簡易モデルを用いる。

```txt
b_y_after = b_y_initial + ΔV_y * t_go / 1000
b_z_after = b_z_initial + ΔV_z * t_go / 1000
```

## FBC-DEC-003：当日の利用形態

標準運用は一人一台とする。ただし、端末不足時は2〜3人で1台、通信・端末トラブル時は講師PC投影による代表操作でも成立するようにする。

ログイン、チーム名入力、サーバー集計、ランキングはv0.1では実装しない。

## FBC-DEC-004：勝利条件・評価軸

100点満点のミッションスコアを用いる。スコア内訳は以下とする。

- 観測スコア：40点
- 安全スコア：40点
- `ΔV` 効率：10点
- 照準バランス：10点

高得点を目的にするだけでなく、観測と安全のバランスを説明できる作戦を重視する。

## FBC-DEC-005：失敗の扱い

失敗を二値で扱わない。危険領域に入った場合は `危険領域` または `安全条件未達` と表示するが、破壊的・揶揄的な表現は使わない。

作戦タイプとして、バランス型、観測重視型、安全重視型、攻めすぎ型、遠すぎ型、調整不足型、危険領域を用いる。

## FBC-DEC-006：実ミッション再現度

本アプリは、はやぶさ2・トリフネフライバイを題材にするが、実ミッション再現ソフトではない。相対速度約5 km/s級、小惑星サイズ約500 m級などの代表値は参考にするが、軌道・スコア・危険判定は教育用簡略モデルとする。

## FBC-DEC-007：公式性・権利まわり

アプリ名は `Flyby Control Challenge` とする。副題は `はやぶさ2・トリフネフライバイを題材にした非公式教育シミュレーション` とする。

JAXA/ISAS/はやぶさ2プロジェクトの公式・公認・監修コンテンツではないことを明記する。公式ロゴ・公式画像はアプリ内素材として使わない。

## FBC-DEC-008：団員に操作させるパラメータ

v0.1で直接操作させるパラメータは、横方向の速度調整 `ΔV_y` と縦方向の速度調整 `ΔV_z` の2つのみとする。修正タイミング、観測開始時刻、観測機器、誤差レベルは団員向け操作UIには出さない。

## FBC-DEC-009：Bプレーン説明・表示

`Bプレーン` という用語は明示的に使う。説明は `小惑星の近くをどのあたりで通るかを見る地図` とする。

Bプレーン画面には、トリフネ中心、危険領域、近いが注意、おすすめ観測領域、安全だが遠い領域、修正前予測点、`ΔV` 後予測点を表示する。

## FBC-DEC-010：ゲームトーン

管制室風をベースにした教育用シミュレータゲームとする。用語は `ミッション開始`、`ΔVを調整`、`Bプレーン予測`、`ミッションスコア`、`作戦タイプ`、`安全余裕` などを用いる。

爆発、墜落、ゲームオーバー、はやぶさ2消失、ギリギリを攻めろ等の表現は避ける。

## FBC-DEC-011：数値表示の粒度

標準表示と詳細表示を分ける。標準表示では、ミッションスコア、作戦タイプ、最近接距離、安全余裕、`ΔV` 合計、観測スコア、安全スコア、作戦コメントを表示する。

詳細表示では、`ΔV_y`、`ΔV_z`、Bプレーン座標、`ΔV` 効率、照準バランス、`T_go = 12 h`、簡易モデル説明を表示する。

## FBC-DEC-012：フライバイ再生画面

簡易フライバイ再生は標準完成目標とするが、最低完成条件ではない。時間がある場合、`T-300 s` から `T+60 s` 程度の簡易再生を入れる。

モデルは次とする。

```txt
x(t) = v_inf * t
r(t) = sqrt(x(t)^2 + d_CA^2)
```

## FBC-DEC-013：視野内／視野外判定

v0.1では厳密なカメラ視野判定は実装しない。距離に基づく観測状態を用いる。

- `r(t) <= 10 km`：高解像度観測域
- `10 km < r(t) <= 30 km`：観測可能域
- `30 km < r(t)`：遠距離

## FBC-DEC-014：スコア表示形式

結果画面では、ミッションスコアを大きく表示し、その下に作戦タイプ、主要値、スコア内訳、作戦コメント、詳細表示を置く。

S/A/B/Cランクは主表示にしない。観測スコアと安全スコアの両方を見せる。

## FBC-DEC-015：チーム間比較・ランキング

v0.1では、ランキング、サーバー集計、リアルタイムリーダーボード、チーム名入力は実装しない。

結果カードはスクリーンショットや投影で共有しやすくするが、比較は任意とする。

## FBC-DEC-016：観測機器の扱い

v0.1では観測機器選択を実装しない。単一の仮想観測カメラとして扱い、観測状態は距離に基づいて評価する。実際の複数機器、視野角、姿勢制御は補足説明に回す。

## FBC-DEC-017：姿勢制御モデル

v0.1では姿勢制御、カメラ指向、LOS追尾、クォータニオン、角速度制限はアプリ内モデルに入れない。最後の補足で、実際は姿勢制御・画像フィードバックも重要であることを説明する。

## FBC-DEC-018：2D表示／3D表示

v0.1の主表示は2D中心とする。Bプレーン表示、`ΔV` 操作、スコア表示、結果カードは2Dで実装する。

時間がある場合の拡張として、フライバイ再生画面の最近接付近だけを簡易3Dまたは疑似3D演出にすることは許容する。本格3D軌道シミュレータ化はv0.1では目指さない。

## FBC-DEC-019：実軌道データ・外部API

v0.1では、実軌道データ、SPICE、Horizons、外部API、リアルタイム天体暦連携は使用しない。固定パラメータの簡略モデルで実装する。

## FBC-DEC-020：難易度設定

v0.1では団員向けUIに難易度選択を表示しない。標準シナリオ1本とする。ただし、実装上は `missionPreset` としてパラメータをまとめ、将来の複数プリセット拡張に備える。

当日の難易度調整は、講師・メンターが提示する口頭チャレンジで行う。

## FBC-DEC-021：メンター用モード

v0.1では独立したメンター用モードを実装しない。結果カード、作戦タイプ、スコア内訳、作戦コメントをメンターが説明に使える形にする。

メンター支援は別途チートシートで行う。

## FBC-DEC-022：主対象デバイス

スマートフォン縦画面を主対象とする。スマホで操作・表示が成立することを最低条件とし、タブレット、PC、講師PC投影にもレスポンシブ対応する。

## FBC-DEC-023：オフライン／低通信環境への対応

完全静的Webアプリとして実装する。GitHub Pagesで配信するが、計算・表示は端末内で完結する。外部API、CDN、外部フォント、外部画像に依存しない。

当日は講師PCにビルド済みファイルまたはZIPをローカルバックアップとして保存する。

## FBC-DEC-024：参考資料・免責表示

アプリ内、README、スライド、メンター資料に、それぞれ粒度を分けて参考資料・免責表示を入れる。操作画面には短い注記のみ、READMEとスライドには非公式教材であることと参考資料を明記する。

## FBC-DEC-025：スライド導入方針

導入は、はやぶさ2紹介、トリフネフライバイ、フライバイの難しさ、`ΔV`、Bプレーン、アプリ操作の順にする。導入は10〜15分程度に抑え、姿勢制御・画像フィードバック等の発展的話題は最後に回す。

## FBC-DEC-026：公式Webアプリ・既存可視化と自作アプリの役割分担

既存可視化は「見る教材」、自作アプリは「考える・操作する教材」とする。自作アプリ内に公式ページや既存可視化をiframe等で埋め込まない。参考リンクとして扱う。

## FBC-DEC-027：ゲームと実ミッションの違いを説明するタイミング

導入では短く「教育用簡略モデル」と説明し、アプリ体験中は `ΔV` とBプレーン操作に集中させる。体験後に「実際はもっと難しい」補足として、姿勢制御、画像フィードバック、観測機器、通信・電力方向、大学での研究に接続する。

## FBC-DEC-028：当日の時間配分

短い導入、2ラウンド制、任意共有、最後の補足で進行する。Round 1は自由操作、Round 2は条件つきチャレンジとする。

## FBC-DEC-029：団員向けワークシート

A4 1枚程度の軽いワークシートを用意する。目的は計算問題や採点ではなく、`ΔV` とBプレーンを使って考えた作戦を記録することである。記入値は結果カードと一致させる。

## FBC-DEC-030：メンター向け補助資料

A4 1〜2枚程度のメンター向けチートシートを用意する。今日のゴール、基本の声かけ、作戦タイプ別の声かけ、実ミッションとの差分、避けたい説明、トラブル時の運用を記載する。
