# SPEC.md

# Flyby Control Challenge v0.1 実装仕様

## 1. 概要

`Flyby Control Challenge` は、はやぶさ2・トリフネフライバイを題材にした非公式教育用Webアプリである。団員は `ΔV_y` と `ΔV_z` を操作し、Bプレーン上の通過点を動かしながら、安全でよく観測できるフライバイ作戦を考える。

本仕様はv0.1の実装対象を定義する。

## 2. 技術スタック

- React
- Vite
- TypeScript
- CSS Modulesまたは通常CSS
- SVGまたはCanvasによる2D描画
- GitHub Pages配信
- 完全静的Webアプリ

外部API、CDN、外部フォント、外部画像読み込み、サーバー通信は使わない。

## 5. 入力

団員が直接操作する入力は2つのみ。

- `ΔV_y`：横方向の速度調整 `[m/s]`
- `ΔV_z`：縦方向の速度調整 `[m/s]`

第1回修正の範囲はそれぞれ `-0.20 m/s` から `+0.20 m/s`、ステップは `0.01 m/s` とする。第2回修正は精密操作として、範囲を `-0.10 m/s` から `+0.10 m/s`、ステップを `0.005 m/s` とする。

v0.1では以下を操作させない。

- TCMタイミング
- 観測開始時刻
- 観測機器
- カメラ向き
- 誤差レベル
- 難易度

## 6. Bプレーン計算

```ts
bYAfterKm = initialBYKm + deltaVYMps * tGoSec / 1000;
bZAfterKm = initialBZKm + deltaVZMps * tGoSec / 1000;
dCAKm = Math.sqrt(bYAfterKm ** 2 + bZAfterKm ** 2);
deltaVTotalMps = Math.sqrt(deltaVYMps ** 2 + deltaVZMps ** 2);
safetyMarginKm = dCAKm - dangerRadiusKm;
```

`dCAKm` が `dangerRadiusKm` 未満の場合、安全条件未達とする。

## 7. 観測状態

厳密な視野判定は行わず、距離に基づく観測状態を用いる。

```txt
r(t) <= 10 km           高解像度観測域
10 km < r(t) <= 30 km   観測可能域
30 km < r(t)            遠距離
```

## 8. フライバイ再生

標準完成目標とする。最低完成条件ではない。

簡易モデル：

```ts
xKm = vInfKmPerSec * tSec;
rKm = Math.sqrt(xKm ** 2 + dCAKm ** 2);
```

推奨再生範囲：

- `t = -300 s` から `t = +60 s`
- 現在時刻、トリフネからの距離、観測状態を表示する。

v0.1では2D表示でよい。時間がある場合のみ、最近接付近を簡易3D・疑似3D演出にする。

## 9. スコア計算

合計100点。

```txt
現在の配点は、17.15「最終分布のみのミッションスコア」に従う。
```

### 9.1 最終安全

最終分布の160点それぞれについて、危険円から3 kmまでS字状に増加する安全ポテンシャルを評価し、その平均を最大50点の安全得点とする。安全ポテンシャルは約1.9 kmに変曲点を持つ。最終3σ楕円が危険エリアへ重なる場合は、合計点を最大59点に制限する。

### 9.2 科学的意義

最終分布の160点それぞれについて、中心距離1 km付近を頂点とし、遠ざかるほど滑らかに低下する科学的意義ポテンシャルを評価する。その平均を最大50点の科学的意義得点とする。安全ゲートは掛けず、安全得点と反対方向へ動く独立した評価とする。

### 9.3 操作過程の扱い

1回目の接近量、2回目による改善量、残ったΔV量は採点しない。2回目の前後における科学的意義・安全性・予想範囲の変化は、結果画面へ参考情報として表示する。

## 17. v0.2 試作：2段階修正と予測分布

活動時間と作戦検討の深さを増やすため、次の教育用モデルを試作する。

- 軌道修正は `T-24 h` と `T-6 h` の2回。
- 2回で使用できる総ΔVバジェットは `0.30 m/s`。
- 使用量は、各回のΔVベクトル長の合計とする。
- 第1回修正後は広い予測分布を表示する。
- 第2回修正前に観測更新が入り、予測分布が狭くなる。
- 最終結果は中心点だけでなく、危険領域へ入る確率と、おすすめ観測領域へ入る確率で評価する。
- 分布は同じ作戦なら同じ結果になる決定的な教育用点群とする。

総ΔVバジェットを超える入力は許可せず、操作中のΔVを残量内へ制限する。UIには使用量と残量を常時表示する。

### 17.1 運用画面フロー

2回のΔVを同時に計画するUIではなく、運用の進行を体験する3画面構成とする。

```txt
第1回運用 T-24 h
  ↓ ΔV実行
追加観測・予測分布更新
  ↓
第2回運用 T-6 h
  ↓ 最終ΔV実行
運用結果・成績評価
```

- 第1回運用画面では、第1回ΔV、広い予測分布、総ΔV残量を表示する。
- 第2回運用画面では、第1回ΔVを確定済みとして扱い、観測更新後の分布、最終確率、残りΔVを表示する。
- 運用結果画面で初めてミッションスコア、達成したこと、改善候補、スコア内訳を表示する。
- 各運用画面から前段階へ戻り、運用をやり直せるようにする。
- 将来は第2回運用と結果画面の間にフライバイ再生画面を挿入する。

### 17.2 感度行列と誤差共分散

各軌道修正時刻に、ΔVからBプレーン座標への2×2感度行列を設定する。

```txt
ΔB_i = H_i ΔV_i
P_Bi = H_i P_ΔVi H_i^T
```

`P_ΔVi` は、ΔV方向の実行誤差と直交方向の実行誤差から作る。誤差の標準偏差は固定下限を持ち、ΔVノルムに比例して増加する。このため、大きなΔVは通過位置を大きく移動できる一方、予測分布も広げる。

追加観測後は、第1回ΔV実行誤差の影響を縮小して最終共分散へ引き継ぐ。第2回ΔVによる実行誤差は最終共分散へ追加する。

第2回運用UIでは、修正なしの場合と現在の入力を比較し、次を表示する。

- Bプレーン中心位置の移動量
- おすすめ観測領域へ入る確率の変化
- 危険領域へ入る確率の変化
- 予測誤差楕円面積の変化

`ΔV_y` と `ΔV_z` のスライダーは、端末幅によらず上下配置とする。

### 17.3 Bプレーン表示

- 第1回・第2回の両運用画面で、ΔV実行前後の3σ予測分布を表示する。
- 分布は中心ほど濃く、3σ外縁へ向かって連続的に透明になる楕円グラデーションとする。
- 安全判定に使う3σ外縁を、分布色に対応する細い破線で重ねて表示する。
- 楕円面積と、実行前後の面積倍率を表示する。
- `0.5 km`、`1 km`、`2 km`、`5 km`、`10 km`、`30 km` の距離圏を表示する。
- 表示範囲を `全体`、`12 km圏`、`4 km圏` で切り替えられるようにする。
- ズームは距離比例関係を維持する線形表示とし、非線形な歪み表示は使用しない。

この試作では、ΔV実行誤差と修正量の関係を見やすくするため、航法誤差は小さな基礎幅として扱い、ΔV実行誤差を強調する。実際の運用では航法誤差、観測誤差、モデル誤差なども重要であることを明記する。

### 17.4 2回の修正の役割と安全評価

内部パラメータは、第1回修正が接近、第2回修正が安全確保として機能するよう調整する。

- 第1回修正後の中心距離が `10 km` 圏内に入ることを接近運用として評価する。
- 第1回修正後、光学航法で判明した中心位置を、第1回修正後の3σ予測分布からランダムに決める。
- 観測結果は第1回ΔVを実行した時点で抽選し、第2回操作中は固定する。
- 第2回修正では、更新後の予測楕円全体を危険領域から外すことを重視する。

最終安全余裕は次の簡易式で評価する。

```txt
3σ安全余裕 = 回転した3σ楕円外縁とトリフネ中心の最短距離 - 危険領域半径
```

`3σ安全余裕 >= 0` であれば、3σ予測楕円全体が危険領域の外にあると扱う。

画面上は予測楕円のみを表示し、確率計算に用いる内部点群は表示しない。

### 17.5 公開値を参考にした距離帯と実行誤差

- トリフネの等価半径を `0.225 km`、長軸方向を含む最大半径目安を `0.42 km` とする。
- 安全バッファ込みの危険領域半径を `0.75 km` とする。
- `1〜3 km` を高解像度観測帯、`3〜10 km` を安全寄り観測帯とする。
- 表面からの距離目安は `中心距離 - 0.42 km` で表示する。
- ΔV実行誤差は、固定下限、ΔVノルム比例項、大きな噴射に対する二次項で計算する。
- 第2回修正の比例誤差率は第1回修正より小さくし、遅い修正の精密さを表現する。
- 操作画面に、現在の入力から見積もった噴射のずれの目安 `3σ` を表示する。

これらは公開情報を参考にした教育用パラメータであり、実ミッションの確定値や実機性能を表すものではない。

### 17.6 第2回運用の楕円表示と精密目盛

- 第2回運用画面には、第1回ΔV後、追加観測後、第2回ΔV後の3つの3σ予測楕円を表示する。
- 追加観測前後の3σ楕円面積と縮小率を表示する。
- 第1回ΔVの各軸操作範囲は `±0.20 m/s`、最小目盛は `0.01 m/s` とする。
- 第2回ΔVの各軸操作範囲は `±0.10 m/s`、最小目盛は `0.005 m/s` とする。
- 第2回修正は、第1回より狭い操作範囲と細かい目盛を持つ精密修正として扱う。

### 17.7 3σ統一と画面の言葉

- 描画する予測分布の外側、楕円面積、3σ安全余裕は3σで統一する。安全スコアは分布標本の距離ポテンシャル平均で計算する。
- 3σは「ほとんどの通り道が入る予想範囲」と団員向けに説明する。
- 操作中は「予測分布」より「通り道の予想範囲」、「実行誤差」より「噴射のずれ」を優先する。
- `Bプレーン`、`ΔV`、`3σ`などの専門用語は残してよいが、直後に平易な説明を付ける。
- 小学生がPC画面を少し離れて見ても読めるよう、説明文・凡例・操作値・結果表示の文字を大きくする。

### 17.8 距離エリア凡例と安全判定

- Bプレーン内には距離エリア名を直接表示せず、マップ下の色付き凡例へまとめる。
- 凡例には、トリフネ本体、赤い危険エリア、1 km、1〜3 km、3〜10 km、30 kmの意味を表示する。
- 安全判定は、3σ楕円の長軸半径による保守的な近似ではなく、回転した楕円外縁と危険円の実際の最短距離を用いる。
- グラデーション表示の外側と破線は3σ形状に一致させる。最終的な安全確認には、破線と画面上の3σ安全余裕を併用する。

### 17.9 旧スコア設計

第2回修正の改善量を独立して20点とした旧スコア設計は、遠方を安全に通る作戦が高得点になりやすいため廃止した。現在のスコアは17.11に従う。

### 17.10 公開情報を参考にした予想楕円

公開資料に示されたトリフネ位置不確かさ `14.6 km` は、Bプレーン上の3σ長半径・短半径としては扱わない。教育用の縮尺と方向差を持つ基礎共分散を使用する。

```txt
第1回修正前の3σ予想楕円：長い方向 7.0 km / 短い方向 3.5 km
追加観測後の3σ予想楕円：長い方向 2.0 km / 短い方向 0.8 km
```

第1回修正後は第1回ΔV実行誤差を加え、追加観測後は観測更新後の基礎共分散と保持された第1回ΔV実行誤差を使う。最終結果では、さらに第2回ΔV実行誤差を加える。このため、最終楕円は第2回ΔVが大きいほど広がる。

追加観測後へ保持する第1回ΔV実行誤差共分散は `3%` とする。これにより、第1回修正後の広い楕円が追加観測で明確に縮小し、第2回修正で安全を仕上げる余地を作る。

### 17.11 段階型ミッションスコア

```txt
1回目で中心近くへ照準   25点
3〜5 kmの観測コース     25点
安全に通れる            30点
2回目で安全へ外した     15点
ΔVを残せた               5点
```

- 第1回修正後の予測中心が中心から0〜2 kmなら照準25点とする。2 kmを超えると段階的に減点し、10 km以上は0点とする。
- 最終中心距離3〜5 kmを観測コース25点とする。近すぎる場合と10 kmを超える場合は減点する。
- 3σ安全余裕が0 km以上なら、予想範囲全体を危険エリアから外せた明確な成功として安全30点を与える。
- 第2回修正によって3σ安全余裕を改善した場合に「2回目で安全へ外した」を加点する。追加観測後に危険だった予想楕円を安全へ外した場合は10点以上とする。
- 最終3σ楕円が危険エリアへ重なる場合、合計点を最大59点に制限する。

この配点は、合理的な段階接近作戦を大きく減点するため、17.12で置き換え、最終的に17.15で廃止した。

### 17.12 成果中心のミッションスコア

この方式は操作過程によって同じ最終分布の得点が変わるため、17.15で置き換えた。

```txt
1回目で初期位置より近づけた       10点
科学的意義得点                    35点
安全得点                          35点
2回目で分布全体の成果を改善       15点

```

- 1回目の接近点は小さな過程評価とし、中心近くを経由しないことを大きく減点しない。
- 最終分布の160標本へ距離ポテンシャルを適用し、観測しやすさと安全さの期待値を合計70点で評価する。
- 第2回修正は、科学的意義得点と安全得点の合計改善量で評価する。
- 点数は成果を評価する。方法の違いを単一の作戦タイプへ分類せず、達成項目と改善候補を表示する。
- 最終3σ楕円が危険エリアへ重なる場合、合計点を最大59点に制限する。

### 17.14 ランダム観測結果と期待値スコア

- 追加観測後の中心位置は、第1回修正後の正規分布に従い、3σ範囲内からチャレンジごとに抽選する。
- 同じチャレンジ中は同じ乱数シードを使い、第2回ΔV操作中に観測結果や確率が勝手に変化しないようにする。
- 最終的な科学的意義得点と安全得点は、最終分布の160標本に距離ポテンシャルを適用した平均値で計算する。
- 科学的意義ポテンシャルは1 km付近を最大とし、遠ざかるほど滑らかに低下する。
- 安全ポテンシャルは危険領域から3 kmまでS字状に増加し、約1.9 kmに変曲点を持つ。
- 科学的意義と安全性は独立して反対方向へ動くため、両方を同時に最大化できない。
- 3σ予測分布が危険領域へ重なる場合の59点上限は維持する。

### 17.15 最終分布のみのミッションスコア

```txt
科学的意義得点   50点
安全得点         50点
```

- ミッションスコアは最終分布だけで決める。
- 1回目の接近量、2回目による改善量、残ったΔV量は得点へ加えない。
- 同じ最終分布へ到達した作戦は、操作手順が異なっても同じ得点になる。
- 2回目の前後における科学的意義・安全性・予想範囲の変化は、採点外の参考情報として結果画面へ表示する。
- 科学的意義と安全性は反対方向へ動くため、100点を取れる作戦はない。得点は、この教材で設定した評価条件による比較の目安として扱う。
- 最終3σ楕円が危険領域へ重なる場合の59点上限は維持する。
