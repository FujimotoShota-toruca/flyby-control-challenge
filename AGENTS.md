# AGENTS.md

このリポジトリで作業するcoding agent向けの指示です。

## 最優先方針

このプロジェクトは、宇宙少年団の例会で使用する非公式教育用Webアプリです。実ミッションの厳密再現ではなく、団員が `ΔV` と `Bプレーン` を用いたフライバイ通過位置調整を体験することを目的とします。

実装時は、以下を優先してください。

1. 当日確実に動くこと。
2. スマートフォン縦画面で操作・表示が成立すること。
3. 団員が `ΔV` と `Bプレーン` を理解できること。
4. 観測と安全のトレードオフが見えること。
5. 結果カードを見れば、作戦を説明できること。
6. `npm run build` と主要テストが通ること。

## 技術制約

- React + Vite + TypeScriptを使用する。
- GitHub Pagesで公開可能にする。
- 完全静的Webアプリとして実装する。
- サーバーを使わない。
- 外部APIに依存しない。
- CDN、外部フォント、外部画像読み込みに依存しない。
- SPICE、Horizons、実軌道データ連携はv0.1では実装しない。
- 2D Bプレーン表示を主役にする。
- Three.js等の本格3D表示はv0.1必須ではない。余裕がある場合のみ、フライバイ再生の最近接付近に限定して簡易3D・疑似3D演出を検討してよい。

## v0.1で実装する中心機能

- `ΔV_y` / `ΔV_z` のスライダー。
- `Bプレーン` 2Dマップ。
- 修正前予測点と修正後予測点。
- 最近接距離 `d_CA`。
- 安全余裕。
- 観測スコア、安全スコア、`ΔV` 効率、照準バランス。
- 100点満点のミッションスコア。
- 作戦タイプ。
- 結果カード。
- Aboutまたは簡略説明。
- 非公式教育シミュレーションであることの短い注記。

## v0.1で実装しないもの

- 実軌道再現。
- SPICE/Horizons連携。
- 外部API。
- サーバー集計。
- ログイン。
- ランキング。
- チーム管理。
- メンター専用モード。
- 講師用ダッシュボード。
- 観測機器選択。
- 厳密なカメラFOV判定。
- 姿勢制御モデル。

- 公式ロゴ・公式画像のアプリ内素材利用。

## 用語説明の原則

専門用語は使ってよいですが、直後に団員向けの説明を付けてください。

推奨表現：

- `ΔV`：探査機の速度を少し変えること。
- `Bプレーン`：小惑星の近くをどのあたりで通るかを見る地図。
- `最近接距離`：トリフネに一番近づくときの距離。
- `安全余裕`：危険領域からどのくらい離れているか。

避ける表現：

- 実軌道を再現している。
- JAXA公式アプリ。
- 近ければ近いほど正解。
- 危険領域に入ると爆発する。

## 実装の分割

1. プロジェクト初期化とGitHub Pages用設定。
2. `missionPresets.ts` の固定パラメータ定義。
3. `flybyModel.ts` のBプレーン・最近接距離計算。
4. `scoring.ts` のスコア計算。
5. 2D Bプレーン表示。
6. `ΔV` 操作UI。
7. 結果カード。
8. 簡易フライバイ再生。
9. レスポンシブ調整。
10. テストとビルド確認。

詳細は `docs/SPEC.md` を参照してください。
